# 章节扩写规则

## 摘要扩写

摘要按子问题展开。每个问题至少包含：

```text
问题目标 -> 方法/模型 -> 求解方式 -> 关键结果 -> 价值
```

摘要不能只写“建立模型并求解”，必须写出具体模型名、算法名和关键结果。

## 问题分析扩写

每个问题分析按以下顺序写：

```text
任务本质
输入与输出
约束和难点
模型选择理由
求解方案
验证方案
```

## 模型建立扩写

公式前必须写清变量含义。公式后必须解释每一项：

```text
设 ...
因此 ...
其中 ...
该式说明 ...
```

## 求解算法扩写

算法步骤必须可复现：

```text
Step 1：初始化参数和数据。
Step 2：构造候选解或状态空间。
Step 3：计算目标函数或评价指标。
Step 4：检查约束条件。
Step 5：更新最优解。
Step 6：输出结果、图表和表格。
```

## 结果分析扩写

结果分析不能只列数值。每个表格或图表后至少写：

```text
由表/图可以看出 ...
其中 ...
这说明 ...
因此 ...
```

## 模型评价扩写

模型评价必须真实，不写空话：

- 优点对应本文具体方法。
- 不足说明具体没考虑什么。
- 改进方向给出可执行扩展。
- 推广应用说明能迁移到哪些领域。
