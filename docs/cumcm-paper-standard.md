# CUMCM Paper Standard

本页是 MathModel Skill 的正式论文范式说明。它不替代建模和代码运行，只约束证据门禁通过后的论文结构、章节展开、图表解释和 Word 排版目标。

## 总体结构

正式论文统一采用 `1 / 1.1 / 1.1.1` 编号：

```text
题目
摘要
关键词

1 问题重述
1.1 背景说明
1.2 题目任务
1.3 本文解决思路概述

2 问题分析
2.1 问题一分析
2.2 问题二分析
...

3 模型假设
4 符号说明

5 模型的建立与求解
5.1 问题一模型的建立与求解
5.1.1 建模思路
5.1.2 变量定义与公式推导
5.1.3 求解算法
5.1.4 结果分析
5.1.5 模型检验或灵敏度分析

5.2 问题二模型的建立与求解
...

6 模型检验与灵敏度分析
7 模型评价与推广
8 参考文献
附录
```

## 字数与密度

- 正式稿默认目标为 `18000-25000` 中文字。
- 推荐中心值约 `22000` 中文字。
- 低于 `18000` 的正式格式检查失败。
- 超过 `25000` 只给 warning，但 Agent 应检查是否出现重复、空泛扩写或图表解释堆叠。

## 摘要

摘要目标 `800-1200` 字，必须按子问题展开：

- 第一段概括背景、研究对象和总体目标。
- 每个子问题至少说明模型、算法和关键结果。
- 最后一段说明模型检验、灵敏度、评价和推广。
- 关键词 `4-6` 个，覆盖模型方法、算法和问题领域。

## 正文章节

问题重述不能复制题面，应压缩背景、提炼规则、按编号说明任务。问题分析必须逐问写清任务本质、变量和约束、模型选择理由、求解思路、预计输出和验证方式。

第 5 章是正式论文的核心。每个 `5.x` 小节必须包含：

- 建模思路。
- 变量定义与公式推导。
- 目标函数、决策函数或约束条件。
- 求解算法，使用 `Step 1`、`Step 2` 等形式。
- 结果表、结果图、图表解释和结论回扣。
- 模型检验、灵敏度分析或合理性分析。

## 图表、公式与算法

- 每张图表必须先在正文中引用，再插入图表，图表后必须有解释段。
- 图题格式建议为 `图5.1.1 xxx`，表题格式建议为 `表5.1.1 xxx`。
- 公式前必须定义变量，公式后必须解释每一项含义，并说明公式如何进入求解。
- 算法不能只写“使用 Python 求解”，必须写清输入、状态变量、遍历或优化步骤、停止条件和输出。

## 评价、参考文献与附录

模型评价必须包含优点、不足、改进方向和推广应用。参考文献建议 `5-8` 条，至少覆盖数学建模、概率统计、运筹优化、算法或题目相关领域。附录应说明代码位置、结果契约、图表表格索引，必要时给出关键代码片段。
