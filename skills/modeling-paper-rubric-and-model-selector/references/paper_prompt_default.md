# 默认论文结构提示词（CUMCM风格）

## 一、摘要使用
【角色设定】你是一位专注于数学建模竞赛（CUMCM）的资深评审专家和学术写作顾问。请你根据我提供的【论文信息】和【写作要求】，撰写一份逻辑严密、学术性强、语言优美的论文摘要。

【论文基本信息】
- 论文题目：[在此处填入论文题目]
- 主要工具：MATLAB, Python
- 问题一关键点：[模型名称]，[算法]，[核心结果/结论]
- 问题二关键点：[模型名称]，[算法]，[核心结果/结论]
- 问题三关键点：[模型名称]，[算法]，[核心结果/结论]

【写作要求】
1. 字数要求：总字数控制在 800 字左右。
2. 语言风格：学术、客观、精炼，多用被动语态（如“建立了…模型”，“采用了…算法”）。
3. 核心强调：必须明确提及软件（MATLAB/Python）与具体算法，突出创新性与结果有效性。
4. 格式结构：严格按六段式结构撰写。

【摘要结构模板】
- 第一段（背景与意义，约50-100字）：背景、核心问题、价值。
- 第二段（模型体系总述，约100字）：三问分别建立的模型概览。
- 第三段（问题一详细求解，约150-200字）：模型→软件→算法→变量→结果。
- 第四段（问题二详细求解，约150-200字）：强调递进或改进点。
- 第五段（问题三详细求解，约150-200字）：模型与指标、目标达成。
- 第六段（总结与展望，约50-100字）：优缺点与改进方向。

## 二、问题重述
【角色设定】你继续担任数学建模竞赛的资深指导教师。请根据【原始题目描述】撰写论文第一章：问题重述。

【写作要求】
1.1 问题背景（200-400字）：漏斗式（宏观→中观→微观），宏大叙事与具体数据结合。
1.2 问题提出：严禁改变题意，学术化润色；用有序列表逐问重述，并暗示递进关系。

## 三、问题分析
【角色设定】你继续担任数学建模竞赛的资深指导教师。撰写论文第二章：问题分析（约1000字）。

【核心原则】
- 宏观剖析，不是步骤罗列；术语专业、逻辑清晰；不提前写具体结果数值。

【写作结构与要求】
2.1 问题一分析：本质判定→数据/因素→理论依据→技术预告（具体模型名与特征）。
2.2 问题二分析：与问题一关系→新增难点→策略调整→体系预告（具体模型名与优势）。
2.3 问题三分析：综合视角→评价维度→方法论证→预期目标（稳健性/普适建议）。

## 四、模型假设
【写作要求】
- 列 6 条关键假设；每条 50-100 字；必须结合题目背景；强调合理性与简洁性。

## 五、符号说明
【写作原则】
- 使用标准数学符号习惯；覆盖主要变量/参数/索引/集合；单位必须标注；符号一致性严格执行。

【输出格式】
1) Markdown 符号表：Symbol | Definition | Unit（符号用 LaTeX）。
2) 纯文本对照清单：汉字 --- 符号。

## 六、模型建立与求解
【总要求】逻辑严密、公式规范（LaTeX）、图表丰富（提示插入位置）、代码可运行。

场景一：机器学习/数据挖掘（预测/分类/回归）
- 数据预处理：缺失、异常、归一化/标准化（含公式）
- 特征工程：提取、筛选/降维（含可视化说明）
- 模型构建：原理、结构、指标
- 训练求解：划分、参数、结果图与分析
- 模型检验：过拟合与残差分析

场景二：评价/决策分析（选优/排序/权重）
- 指标体系：维度、指标解释、正负向、一致化公式
- 权重确定：方法与核心步骤公式、权重表与意义分析
- 评价求解：原理、得分公式、排序与原因分析
- 灵敏度：权重扰动与稳定性结论

场景三：优化/规划（路径/调度/排班）
- 决策变量：类型与含义
- 目标函数：与题意一致（含公式）
- 约束条件：资源/物理/业务（完整方程组）
- 求解算法：选择依据、编码策略、流程
- 结果分析：最优方案、与基准对比、提升率

## 七、模型评价与推广
【总体要求】800字左右，客观中肯，优缺点对应模型细节，不说套话。

6.1 优点（3-4点）：结合对比实验或指标改进描述。
6.2 缺点（2-3点）：白璧微瑕，不否定核心价值。
6.3 改进：针对缺点给出技术性方案。
6.4 推广：横向迁移 + 纵向深化 + 社会/经济价值。
